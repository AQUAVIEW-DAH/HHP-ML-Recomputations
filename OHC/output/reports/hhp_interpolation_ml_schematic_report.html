3. TEOS-10 / GSW OHC and TCHP Calculation

3.1 Core software package

  • gsw: Python Gibbs SeaWater (GSW) implementation of TEOS-10.
  • numpy: profile cleaning, interpolation, and trapezoidal integration.

3.2 Inputs

InputMeaning
verticalEither pressure in dbar (Argo-style) or depth in meters (RTOFS-style).
temp_cIn-situ temperature profile in degrees Celsius.
salinity_psuPractical salinity profile.
lat, lonUsed in TEOS-10 conversions between pressure, depth, and absolute salinity.

3.3 Main constants and formulas

ParameterValueHow it is used
REF_TEMP_C26.0Reference isotherm for D26 and OHC/TCHP.
J_PER_M2_TO_KJ_PER_CM21 / 1e7Converts integrated heat from J/m² to kJ/cm².

D26 rule. compute_d26() returns the depth where the profile crosses downward through 26 C. If the surface temperature is already below 26 C, the function returns None immediately.

For valid warm profiles, the code:

  • converts pressure↔depth using gsw.z_from_p or gsw.p_from_z,
  • interpolates the profile to the exact 26 C crossing depth,
  • converts practical salinity to absolute salinity with gsw.SA_from_SP,
  • computes density and heat capacity with gsw.rho_t_exact and gsw.cp_t_exact,
  • integrates heat excess (T - 26 C) * rho * cp from the surface to D26 using np.trapezoid.

3.4 Why only warm-water regions show valid OHC/TCHP

Raw Argo observations are global. The derived variables D26 and TCHP are not. A point only contributes to the OHC/TCHP maps if the upper ocean actually crosses 26 C.

2024 seasonal subsetRaw Argo pointsRaw points with |lat| > 45Valid D26/TCHP pointsValid points with |lat| > 45
Winter (JFM)38,69310,80411,1330
Summer (JAS)41,80411,45614,2770

Interpretation: there are many Argo profiles outside the equatorial and subtropical warm belt, but they usually do not have surface temperatures above 26 C and therefore never produce a valid D26 crossing. This is why the final OHC/TCHP maps concentrate in warmer ocean regions.

4.2 Exact-location collocation

RTOFS is not compared on its native grid against a separately gridded Argo surface. Instead, RTOFS is first sampled at the exact Argo locations.

ParameterValueCode effect
K_NEIGHBORS8Uses the 8 nearest native RTOFS grid cells around each Argo point.
Distance spaceUnit-sphere xyz from lat/loncKDTree queries nearest cells after converting lat/lon to 3D unit vectors.
Point interpolationInverse-distance-squaredweight = 1 / distance^2, with exact-hit safeguard at distance <= 1e-6.
Argo point (lat, lon, date)
   |
   v
Find 8 nearest RTOFS cells on native grid using cKDTree in xyz space
   |
   v
Compute great-circle distance from chord distance on unit sphere
   |
   v
If exact grid hit exists -> use exact value
Else -> weighted mean with 1 / distance^2

4.4 The 100 km mask

The 100 km mask is a support mask, not a smoothing parameter. It prevents the code from showing values where no collocated observations are sufficiently nearby.

ParameterValueDefinition in code
MASK_DISTANCE_KM100.0Grid nodes farther than 100 km from the nearest collocated observation are set to NaN.
GRID_RES_DEG0.25Rendered global grid cell centers every 0.25°.

The code converts kilometers to unit-sphere chord length:

chord_from_km(distance_km) = 2 * sin((distance_km / EarthRadiusKm) / 2)

That chord threshold is then compared with nearest-neighbor distances in xyz space.

  • Gaussian: grid nodes outside 100 km are skipped after the local-neighbor search.
  • LinearND / RBF: the field is computed first, then any node whose nearest observation is beyond 100 km is set to NaN.

Interpretation of 100 km. The mask is there to keep the map honest. It says “only show the interpolated value if at least one real collocated observation is within 100 km.” It does not define how much the method smooths; that role belongs to the Gaussian length scale or RBF epsilon.

5. Machine-Learning Correction Schematic

5.1 Training-table design

The ML project does not train on rendered images. It trains on the multiyear collocated point table:

  • OHC/output/ml_collocation/data/argo_rtofs_collocated_2024_2025.parquet
  • Total collocated rows: 41,347
  • Total dates: 90
  • Finite residual rows per target: 12,543

5.2 Residual-learning target

delta_tchp = Argo_TCHP - RTOFS_TCHP_at_Argo_point
delta_d26  = Argo_D26  - RTOFS_D26_at_Argo_point

corrected_prediction = raw_RTOFS + predicted_delta

This keeps the model focused on correcting RTOFS rather than rebuilding Argo from scratch.

5.5 Locked evaluation protocol

The official benchmark note is frozen in OHC/output/ml_benchmarks/locked_protocol_note.md.

  • Split unit: date
  • Split type: forward-chaining blocked validation
  • Embargo: 1 date
  • Folds: 3
  • Random row splitting: explicitly forbidden

This was chosen to reduce leakage from same-day spatial clustering and nearby profiles from the same ocean state appearing in both training and validation.

9. Practical Interpretation for Mentoring / Review

  • The warm-water restriction in the final maps is a property of the 26 C isotherm definition, not a lack of raw Argo coverage.
  • The 100 km mask is a visualization-support threshold to avoid showing interpolation too far from actual observations.
  • The machine-learning correction is currently a point-collocated residual model, not a gridded deep-learning model.
  • The current strongest operational path is the year-holdout XGBoost correction, trained on 2024 and tested on 2025.
